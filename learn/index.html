What Makes It Tick

Six layers working together, fully automated

VPN Kill Switch

All download traffic flows through a WireGuard tunnel. If the VPN drops, iptables blocks everything — your real IP is never exposed. The tunnel auto-restarts within seconds.

Smart Search

Get Started

From zero to streaming in minutes

Clone the Repository

Get the code onto your server (NAS, VPS, or any Docker-capable machine).

git clone https://github.com/TrashedPandai/arr-barr.git && cd arr-barr

Run Setup

The setup wizard handles everything: installs the CLI, creates your environment file, builds the directory tree, applies config templates, and pulls all Docker images.

bash .cli/setup.sh

You'll be prompted for your VPN credentials (WireGuard private key, public key, and endpoint). Get these from your VPN provider.

Start the Stack

Bring everything online. Gluetun establishes the VPN tunnel first, then download clients start, then all other services launch in parallel.

arr start

Verify Everything Is Healthy

Run the health check to hit all 15 service endpoints in parallel. Green dots mean healthy.

arr health

Configure Prowlarr

Open Prowlarr at :9696 and add your indexers. They'll automatically sync to Radarr, Sonarr, and Lidarr. Add FlareSolverr as an indexer proxy for Cloudflare-protected sites.

Start Requesting

Open Jellyseerr at :5055, sign in with your Jellyfin account, and start requesting movies and shows. They'll be searched, downloaded, organized, and added to Jellyfin automatically.

How It All Connects

🎥 Requesting a Movie or Show

▼

1. Open Jellyseerr (:5055) and search for what you want.

2. Click Request. Jellyseerr adds it to Radarr (movies) or Sonarr (TV).

3. The arr app automatically searches all indexers via Prowlarr.

4. Results are scored by custom formats — the best release is grabbed.

5. The download client fetches it through the VPN tunnel.

6. On completion, the file is hardlinked into the library and Jellyfin picks it up.

7. Bazarr automatically fetches subtitles for the new file.

Total time: usually 5–30 minutes depending on the release.

🎵 Adding Music

▼

Open Lidarr (:8686), search for an artist, and add them. Lidarr uses MusicBrainz for metadata and monitors for new releases.

Albums are scored using Davo's custom formats: CD rips and preferred release groups score highest, vinyl rips are rejected.

Music appears in Jellyfin's music library automatically.

📚 Getting Books & Audiobooks

▼

Enable the book profiles in your .env file (COMPOSE_PROFILES=kavita,audiobookshelf), then restart.

Open LazyLibrarian (:5299) and add authors. It searches using GoodReads metadata.

The PostProcessor automatically sorts ebooks to Kavita and audiobooks to Audiobookshelf every 10 minutes.

🎮 Finding Games

▼

Open QuestArr (:5002) and search for a game. Pick a release and it's sent to the download client.

A background script (game-import.sh) runs every 15 minutes to move completed downloads into the game library with 9 safety checks.

🔄 How Upgrades Work

▼

Every 15–30 minutes, arr apps check RSS feeds for new releases. If a release scores higher than your current file (e.g., a WEB Tier 01 replacing a Tier 03), it automatically downloads the upgrade.

The old file is replaced and the new one is hardlinked in. This continues until the cutoff CF score (10,000) is reached — effectively always looking for improvements.

Tips & Tricks

💡 Check VPN Before Anything Else

▼

If downloads aren't working, start with arr vpn. It shows whether the tunnel is up, your public IP through the tunnel, and whether the download clients are reachable.

💡 Use arr doctor Regularly

▼

Doctor self-heals most common issues: missing directories, outdated CLI binary, missing .env variables, wrong permissions. Run it after updates or if something feels off.

💡 VPN Group Restart

▼

arr restart vpn restarts Gluetun, waits 10 seconds for the tunnel to establish, then restarts both download clients. This is the correct order — restarting download clients without the VPN would leave them without network access.

💡 Back Up Before Updates

▼

Run arr backup before arr update. Backups are timestamped tar.gz files of all service configs. Use arr backup --prune 5 to keep only the 5 most recent.

💡 Understanding Hardlinks

▼

A 50 GB movie exists once on disk but has two filenames — one in the download directory (for seeding) and one in the media library (for streaming). When the torrent is removed, only one link is deleted; the library copy persists.

This only works because every service mounts the same root directory (DATA_ROOT:/data). If you use separate mounts, hardlinks break and files get copied instead, doubling disk usage.

Deep Dive

The complete technical architecture

Request to Stream: The Complete Flow

Every piece of media follows this journey. Each step is automated.

1 — Request

User requests content

Via Jellyseerr (browse by poster, click request) or directly in the arr app's UI. The item is added to the wanted list.

2 — Search

All indexers queried in parallel

The arr app sends the search to Prowlarr, which fans out to 12 indexers simultaneously. FlareSolverr handles Cloudflare-protected sites with a headless browser. Results stream back.

3 — Score & Select

Custom formats pick the best release

Each result is scored against up to 48 custom formats. Release group tier, source quality, HDR format, streaming service, and more. Unwanted content (LQ, BR-DISK, AV1) is instantly rejected at -10,000. The highest-scoring release wins.

4 — Download (VPN)

Fetched through encrypted tunnel

The release is sent to Transmission (torrents) or SABnzbd (usenet), both running inside Gluetun's VPN tunnel. All traffic is WireGuard-encrypted. Kill switch active.

5 — Import & Organize

Hardlinked into the library

Torrents are hardlinked (zero extra space, keeps seeding). Usenet files are moved (staging cleared). Files are renamed per TRaSH naming convention with quality tags, release group, and media info.

6 — Stream

Available in Jellyfin

Jellyfin detects the new file, fetches metadata, and makes it available. Bazarr automatically downloads subtitles. The content appears in "Recently Added" and is ready to watch.

VPN Kill Switch

The firewall uses a default DROP policy. Only three types of outbound traffic are allowed:

  • VPN tunnel — all internet traffic must exit through WireGuard
  • Docker internal — container-to-container communication
  • WireGuard handshake — to re-establish the tunnel if it drops

Everything else is silently dropped. IPv6 is fully blocked. If the tunnel goes down, download clients lose all internet access until it recovers (auto-restart enabled).

Hardlink Architecture

The single-mount design that eliminates duplicate disk usage

Downloads

/data/downloads/torrents/
radarr/Movie.mkv

Torrent keeps seeding here

⟶
hardlink
Same inode
Zero extra space
Library

/data/media/movies/
Movie (2024)/Movie.mkv

Jellyfin streams from here

This works because all containers mount DATA_ROOT as /data. Downloads and media are on the same filesystem, so the kernel can create a second filename pointing to the same data blocks. A 50 GB file uses 50 GB total — not 100 GB.

Three Mount Tiers

Tier 1: Full /data

Transmission, SABnzbd, Radarr, Sonarr, Lidarr, Bazarr, LazyLibrarian

Read-write access to both downloads and media. Can hardlink.

Tier 2: Read-only media

Jellyfin, Kavita, Audiobookshelf

Only see their specific media directories. Cannot modify files.

Media Pipelines

Each media type has its own dedicated pipeline

🎥 Movies — Radarr

▼

48 custom formats (TRaSH Guide) with 34 scored. 6 release group tiers, IMAX boosts, movie version detection.

Cutoff: Bluray-1080p. Keeps upgrading toward higher-tier releases.

Naming: Movie Name (2024) [CFs][Quality][Audio][HDR][Video]-Group

Categories: Torrents → radarr/, Usenet → movies/

Post-import: Bazarr auto-fetches subtitles, Jellyfin detects new file.

📺 TV Shows — Sonarr

▼

44 custom formats with streaming service scoring (+75 each for 17 services), HDR/DV boosts, and WEB release group tiers.

Cutoff: WEB 2160p (future-proofed for 4K TV content).

Season packs: When >50% of episodes are missing, prefers downloading entire season.

Anime support: 15 anime series with absolute numbering and language-aware naming.

Categories: Torrents → tv-sonarr/, Usenet → tv/

🎵 Music — Lidarr

▼

5 custom formats (Davo's Guide): CD (+500), Preferred Groups (+500), WEB (+200), Lossless (+100), Vinyl (-10000).

Min CF score: 1 — releases must match at least one positive CF. Prevents grabbing random low-quality uploads.

Cutoff: Lossless (FLAC, ALAC). Accepts lossy initially, upgrades to lossless.

Metadata: MusicBrainz. Artist folder: {Artist NameThe} (sorts "Beatles, The" under B).

📚 Books & Audiobooks — LazyLibrarian

▼

PostProcessor runs every 10 minutes, sorting by file type: ebooks (.epub/.pdf/.mobi) → /media/books/, audio (.mp3/.m4b) → /media/audiobooks/.

Metadata: GoodReads. Generates cover art and .opf metadata sidecars.

Docker mods: Calibre (ebook conversion) + FFmpeg (audio processing).

Readers: Kavita (ebooks, browser-based) and Audiobookshelf (audiobooks, with mobile apps).

🎮 Games — QuestArr

▼

Search-only — no built-in import engine. Uses IGDB for metadata.

Import: game-import.sh runs every 15 minutes via cron, with 9 safety layers: API completion check, settling time, duplicate detection, empty check, lockfile, and more.

Hardlinks: Torrent games are hardlinked, usenet games are moved.
